1. 產品背景與痛點分析

痛點一：基礎設施限制

租賃辦公場域無法取得物聯網 (IoT) 改裝權限，導致傳統需要 API 串接的機器人方案無法落地。

痛點二：人力價值虛耗

專人需頻繁往返樓層領取包裹、等待貨梯，嚴重剝奪行政團隊處理高價值業務的時間。

2. 產品願景與定位

"打造具備高度適應性的免改裝 (Retrofit-Free) 垂直物流解決方案。"

利用 AI 視覺、AMR 自主底盤與 6 自由度機械臂技術，實現「仿生操作」，在完全不改裝現有電梯設施的情況下，完成一站式收發服務。

3. 核心業務邏輯與流程

01

通訊連動與派車

1F 人員發送需求 → 8F FMS 雲端系統接收 → 指派最佳路徑 AMR 出發。

02

垂直運輸搭梯策略

物理性按壓外呼鈕 → 視覺判定車廂人數 → **客滿自動等下一班** → 進入後按壓內部選層鈕。

03

地面自主導航

依據 SLAM 地圖自主移動 → 動態識別路人並執行避障 → 到達目標點執行裝載。

5. 業務營運與技術諮詢重點

物件識別與追蹤

  • 籠車物件需整合 Barcode / QR Code 或 RFID 感應。
  • 可選配載貨量感測器，偵測「倉位飽和度」。

安全認證與簽收

  • 身分對接：支援刷卡、OTP 驗證碼或臉部辨識。
  • 異常處理：收件人不在位時的自動回站或重送邏輯。

場域邊界條件

  • 電梯環境：需確認門檻高度、按鈕型式及 5G 通訊訊號。
  • 自動回充：需規劃專用 Docking Station 電力點位。

系統擴充性

  • 多機協作：考慮電梯口排隊的優先級權重邏輯。
  • 自動門串接：視覺辨識開啟非電梯之感應門。
技術關鍵指標：辨識成功率 > 99% | 按壓精度 ±2mm | 避障反應 < 0.2s
